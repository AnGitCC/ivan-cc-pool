# 填写域名（Cloudflare）与计分编辑补齐设计

## 1. 背景

当前场次的二维码与分享链接来自 `session.publicUrl`，该字段在创建场次时由 `SURVEY_ORIGIN + /s/{slug}` 生成并落库。随后：

- 分享弹窗二维码直接把 `publicUrl` 编码进二维码
- 场次管理页与场次工作台直接展示并复用 `publicUrl`

在预发布阶段，填写页需要通过 Cloudflare 代理的域名对国内访问友好；并且域名切换后，历史场次的 `publicUrl` 仍可能指向旧域名，导致扫码无法打开填写页。

此外还存在两个 UI 缺口：

- 场次管理页“创建场次”按钮在某些宽度下文本被挤压换行
- 问卷编辑器缺少“选项分值”编辑能力，导致手工建卷无法配置计分题

## 2. 目标

1. **二维码/分享链接稳定指向填写域名**：始终使用 Cloudflare 代理域名（或其等价的填写域名配置）打开 `/s/{slug}`。
2. **历史场次可用**：无需重新创建场次，旧场次扫码也能跳转到填写页。
3. **创建场次按钮单行展示**：在横向布局中按钮不被挤压换行。
4. **手工建卷支持计分**：对正式题（单选/多选）提供最小可用的“选项文案 + 分值”编辑入口。

## 3. 非目标

- 不在本轮改变 `rating` 题的计分语义与统计口径。
- 不在本轮引入“问卷变更后对历史答卷重算聚合”的能力。
- 不在本轮引入场次问卷 schema 快照机制（后续单独一轮处理口径漂移问题）。

## 4. 方案概览（推荐）

### 4.1 填写域名：以 `SURVEY_ORIGIN` 为唯一来源

- 继续使用 `buildSurveyUrl(slug) => ${SURVEY_ORIGIN}/s/${slug}` 作为唯一构造方式。
- 将 `SURVEY_ORIGIN` 配置为 Cloudflare 代理域名（填写页域名），确保二维码对国内扫码可用。

### 4.2 历史场次链接修复：基于 slug 回填 `publicUrl`

问题根因是 `publicUrl` 落库后不随环境变量变化而自动更新。

推荐做法：

- 增加一个服务端修复动作：对给定 `sessionId`，读取其 `slug`，计算最新 `buildSurveyUrl(slug)`，并更新 `session.publicUrl`。
- 在以下关键读取路径中执行“懒修复”（只在发现不匹配时触发）：
  - 场次管理页列表渲染前（批量）
  - 场次工作台加载统计数据前（单场次）

判定“不匹配”的规则：

- `session.publicUrl` 不以 `${SURVEY_ORIGIN}/s/` 开头，或
- `session.publicUrl` 与 `buildSurveyUrl(session.slug)` 不一致

这样可以保证：

- 环境变量切换后，旧场次会在首次被访问时自动修复
- 二维码与复制链接会立刻变为正确域名

### 4.3 分享弹窗：仍以 `publicUrl` 为单一事实来源

保持现有组件心智不变：分享弹窗只接收一个 `url` 并生成二维码。修复动作会把 `publicUrl` 回填正确，从而不需要在前端到处改为“临时拼接 URL”。

### 4.4 创建场次按钮换行修复

场次管理页表单在 `md` 断点为横向 flex，输入框使用 `w-full` 导致按钮被挤压换行。

修复策略：

- 输入框改为 `flex-1`（占剩余空间但不强制占满）
- 按钮增加 `shrink-0 whitespace-nowrap`（不收缩、不换行）

### 4.5 问卷编辑器：正式题选项分值编辑（MVP）

当前 schema 已支持两种选项形态：

- 纯文本选项：`string`
- 计分选项：`{ label: string; score: number }`

本轮在问卷编辑器中补齐 UI：

- 仅对“正式题”且题型为 `single` / `multiple` 提供选项编辑区域
- 每个选项包含：
  - 选项文案输入框（label）
  - 分值数字输入框（score）
  - 删除选项按钮
- 提供“新增选项”按钮

保存策略：

- 编辑器草稿序列化为 `draftJson` 提交到现有 `createQuestionnaire/updateQuestionnaire`，后端继续用 `questionnaireSchema.parse` 校验并落库。

## 5. 验收标准

1. 任何场次的分享弹窗：
   - “打开填写页”可正常打开 `/s/{slug}` 页面（不报错）
   - 二维码扫码可打开同一地址
2. 历史场次（在切换 `SURVEY_ORIGIN` 之前创建的场次）：
   - 首次打开场次管理页或场次工作台后，复制链接与二维码都变为新的填写域名
3. 场次管理页创建区：
   - 在桌面端横向布局中，“创建场次”按钮文字不换行
4. 问卷编辑器：
   - 单选/多选题可编辑选项文案与分值，并能保存
   - 保存后新建场次、填写并提交答卷，计分题的 `totalScore` 与统计结果与分值一致

## 6. 风险与取舍

- **懒修复带来的写操作**：访问列表时可能触发对旧场次的 DB 更新；但只在检测到不一致时触发，且次数有限（一次修复后不再触发）。
- **计分口径漂移**：问卷编辑后会影响后续填写与统计；本轮不解决，需要后续通过“场次快照 schema”收口。
